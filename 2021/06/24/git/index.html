git相关技术点

June 24, 2021 3716

基础

  • git add . 提交至暂存区
  • git commit 把文件提交到仓库
  • 或：
  • git commit -m '' 本次提交说明
    为什么要执行两步

    git add 将变更的文件提交至暂存区

    git commit 将暂存区的文件提交至本地仓库

    修改后

  • git status 查看当前仓库状态
  • git diff 查看具体修改了什么，对比差异
  • git add . 提交至暂存区
  • git commit -m '本次修改备注' 提交到本地仓库，添加备注信息
  • git push 推送到远程分支

    查看log日志

  • git log 显示从最近到最远的提交日志
  • git log --pretty=oneline 简化日志界面

    版本回退

  • git reset --hard HEAD^ 退回上一个版本
  • git reset --hard HEAD^^^ 退回三个版本
  • HEAD指向的版本就是当前版本，所以在历史版本中切换，就指定当前HEAD： git reset --hard commit_id
  • 切换前用 git log 查看提交历史，以便确定退回到哪个版本
  • 关闭命令行窗口就找不到被恢复抹掉的版本了调用：git reflog查看命令历史，可以找到之前的commit版本

    git add

  • 提交到暂存区
  • 只有git add 提交到暂存区的才会被git commit 到仓库

    撤销工作区更改

  • git chekout --文件名 撤销更改
  • ‘–’ 两条杠非常重要，如果不加 – 就成了切换分支的命令
  • 撤回到最近一次commit 和add 的状态

    撤销暂存区更改

  • git status 查看状态
  • git reset HEAD <file> 将暂存区的更改撤销，重新放回工作区

    撤销仓库中的更改（未push到远程分支）

  • git reset --hard HEAD^撤回到上个版本

撤销总结：

  • 撤销工作区更改：git chekout --file
  • 撤销暂存区更改：git reset HEAD <file>
  • 撤销仓库更改：git reset --hard HEAD^ 或 git reset --hard commit_id

    删除文件

    当工作区删除某个文件时
  • git status查看当前状态，git会知道有文件被删除
  • git rm 'file' 从版本库中删除或者git add &commit效果是一样的
  • 如果是误删了文件(不慌，版本库里还有)：git chekout --file

远程分支

关联分支

  • 关联一个远程分支git remote add origin git@serve-name:path/repo-name.git
  • 关联后使用git push -u origin master第一次推送master分支的所有内容
  • 之后使用 git push origin master推送最新更改

克隆

  • git clone 地址将代码克隆到本地
  • Git支持多种协议包括https，但通过ssh支持的原生git协议速度更快

创建合并分支

  • 创建dev分支git checkout -b dev
  • 查看分支 git branch 会列出所有分支，带*号的是当前分支
  • 切换分支 git checkout dev
  • 在dev上进行修改并提交，再切回mastergit checkout master发现dev的提交在master上没有
  • 合并分支git merge dev
  • 合并完之后就可以删除dev分支了git checkout -d dev
  • gir branch -D dev(强制删除)
  • 删除后,查看分支git branch 就只剩master了

基于某个分支创建我的分支

  • git fetch 拉取
  • git checkout PC05 切换到PC05分支
  • git checkout -b PC05_pangyongkang 创建我的分支
  • git push origin PC05_pangyongkang推送到远程，创建我的远程分支
  • git pull拉取分支，会提示关联远程分支
  • git branch --set-upstream-to=origin/PC05_pangyongkang PC05_pangyongkang 将本地分支与远程分支关联
  • git pull 就可以拉取了(此时必然没有更新)

查看commit记录

  • 1、仅仅想看最后一次的提交
    git log -n 1

  • 2、想看到最近一次提交所有更改过的文件
    git log -n 1 --stat

  • 3、想看到最近一次提交所有更改的细节
    git log -n 1 -p

并不想提交的代码，但是要切换分支。stash储藏代码

  • git stash 储藏当前代码（清空工作区）
  • git stash list查看储藏的代码list
  • 恢复
  • 方法1：git stash apply 恢复，保留stash中的储藏
  • git stash apply stash@{0} 0 为stash list中的下标
  • git stash drop 删除stash中的记录
  • 方法2：git stash pop 恢复，并删除stash中的储藏
  • 总结（常用）：
  • git stash 储藏
  • git pop 恢复

    标签管理

发布版本时，在版本库中打一个标签(tag)

  • git checkout PC01切换到要标签的分支
  • git tag v1.0 对当前分支打个标签
  • git tag 查看所有标签
  • 默认标签是打在最新提交的commit上的
  • 标签总是和某个commit挂钩。如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签。
  • git tag -d v1.0 删除标签
  • git push origin v1.0将标签推送至远程

配置别名

  • git config --global alias.st status现在敲git st = git status。

查当前本地分支对应哪个分支

  • git branch -vv

    cherry-pick仅合并某个commit提交

  • 先用git log查看commit的id，复制下来

  • git checkout 到master分支下

  • git cherry-pick <C2_id>如果cherry-pick过程没有出现冲突的话，那就是完成了合并

  • 如果有冲突: 1>解决冲突.2>git add . 3>git cherry-pick –continue

commit规范

用于说明 commit 的类别，只允许使用下面7个标识。

```feat：新功能（feature）

fix：修补bug

docs：文档（documentation）

style： 格式（不影响代码运行的变动）

refactor：重构（即不是新增功能，也不是修改bug的代码变动）

improvement：优化

test：增加测试

chore：构建过程或辅助工具的变动```
